git主题讲解

git是一个分布式版本控制工具

如何理解分布式?

结合廖雪峰老师的git笔记（主要看评论和回复）的讲解和自己理解，在这先写下笔记。

首先假设：本地和远程仓库

我们每一次的commit都是一次版本的记录 而这版本的记录是不需要联网的，而且只要commit我们就能在本地了解到我们上上个版本或是上上上个版本更改了什么（这里要注意一下 不需要联网）也就是 项目版本是在我们本地控制，当然如果你不pull别人的代码下来也就不知道别人更改了什么 而集中式（如SVN）每次的版本控制需要联网才能知道，而且在你不经常commit的情况下很难知道自己修改了什么。

各自的优点：

svn：可以清楚知道别人在某某文件修改了什么

git：优化了个人的版本控制，在没有联网的情况下可以清楚了解到自己改了什么，优秀的分支系统

git的主要命令

先一一将命令写出来 然后简单清晰地解释代码的用法

基本：

git init —初始化git(创建文件夹.git)

git add . —添加该目录下所有文件到git仓库 跟踪新文件 放入暂存区

git commit -m “每次提交的注释” —提交代码

git push origin master —将代码上传到远程仓库

查看:

git branch [-a/r] —查看本地所有分支

git status —查看当前状态

git remote —查看已经配置的远程仓库git保存的简写与其对应的URL

git remote -v —查看所有远程仓库

远程仓库：

git remote add -name -url —新建远程仓库

git remote rename -old name -new name —重命名远程仓库

git remote rm -name —删除

git pull -origin name -branch —同步某个远程仓库的某个分支

检查文件差异：

git diff -filename
